# 마운드 심리전 — 타자·투수 태그 확장 기획서

**문서 버전:** v1  
**작성 목적:** 타자·투수 태그 풀을 확장하되, 기존 mind / tendency / boss / memory v1과 **중복·이중 적용 없이** 역할을 나누기 위한 정본 기획  
**관련 문서:** `게임시스템_기획서.md`, `게임기획서_기획자용.md`

---

## 1. 총평 요약

| 항목 | 결론 |
|------|------|
| 제안 20개 타자 태그 | **8개 제외**, **4개 통합·확장**, **8~10개 신규 카탈로그**로 압축 |
| 투수 태그 | 표에 없었음 → 타자 신규와 **1:1 대응**하는 **5개** 추가 후보 |
| 구조 | 타자에 `batterTagCatalog` 신설 (투수 `pitcherTagCatalog`와 동급) |
| UI | 타자 카드 태그 **최대 5개** 유지 — mind 1 + tendency 1 + tag 1~2 + (보스/파생) |

**한 줄:** 태그는 **「구종·코스·카운트 특화」**만 담고, 심리전·패턴·거짓단서·역노림은 **기존 레이어 + memory v1**에 맡긴다.

---

## 2. 역할 분담 (중복 방지 원칙)

| 레이어 | 담당 | 태그로 옮기지 않을 것 |
|--------|------|------------------------|
| **mind** (마음 4종) | 반응 신뢰도, 거짓 단서 확률, patternLearn 배율 | 교활한 반응, 역노림 |
| **tendency** (숨은 성향) | approach, 0-0 / 2S / 볼 기다림 **기본 성향** | 초구 적극(단독), 볼넷 유도, 2스트 생존 |
| **memory v1** | 반복 구종·코스·2S 유인·풀카운트 패턴, 의심 Δ | 패턴 감지 |
| **bossGimmick** | 보스 전용 극단 기믹 | 클러치(보스), 천재타자(보스) |
| **batterTag** (신규) | **구종 계열 × 코스 × 카운트** 조건부 보정 | 위 항목과 효과 겹치면 **태그 쪽 제외** |
| **pitcherTag** (기존+신규) | 투수 빌드·제구·유인·병살 | 타자 약점과 **대칭** 설계 |

**구현 규칙**

- 한 타자: mind 1 + tendency 1 + batterTag **0~2** (일반), 보스는 tag +1 또는 bossGimmick 유지  
- 동일 조건·동일 stat에 두 레이어가 동시에 bonus를 주지 않음  
- `역노림` / `패턴 감지` / `교활한 반응`은 **태그 ID로 추가하지 않음** (시스템·mind 전용)

| 제안 태그 | 제외 사유 | 유지 위치 |
|-----------|-----------|-----------|

| 2스트 생존 | `인내형` foul +0.10, 보스 커트장인 +0.18 | tendency / boss |

| 클러치 타자 | 보스 **클러치** 기믹 | bossGimmick |

## 4. 통합·확장 (4건)

### 4.1 초구형 → 2분할 (tendency 개편)

기존 `firstPitch` 하나를 나눈다. **태그가 아니라 tendency**로 유지.

| ID | UI 라벨 | approach | 0-0 보정 |
|----|---------|----------|----------|
| `firstPitchAggro` | 초구 적극 | 초구 | inZone swing +0.12 |
| `firstPitchWatch` | 초구 관찰 | 신중 | !inZone chase −0.10, inZone swing −0.06 |

→ 제안 **「초구 적극 / 초구 관찰」**은 태그가 아니라 **성향 카드**로 처리.

### 4.2 몸쪽 응징 — `응징형` 확장

| | 기존 slugger | 확장 |
|--|--------------|------|
| 조건 | `centerMistake` (정중앙 실투) | **몸쪽** (`col ≤ 0` 또는 side `inside`) 실투 |
| 효과 | contactQuality +8 | contactQuality +6 (중앙 +8과 **중복 적용 안 함**, 더 유리한 쪽만) |

### 4.3 느린공 참기 — 보스 `눈야구` 일반화

| | 보스 눈야구 | 일반 태그 `offspeed_patience` |
|--|-------------|-------------------------------|
| 조건 | 존 밖 전체 chase −0.12 | **offspeed 계열** 투구 + 존 밖 chase −0.10 |
| 비고 | 보스 유지 | `안구형`과 **동시 적용 시** offspeed만 추가 (flat chase 중첩 cap −0.14) |

### 4.4 바깥쪽 추격

보스 **커트장인**(2S foul +0.18)과 **다른 축**:  
**바깥 코스** (`col ≥ 2` / side `outside`) + 스윙 시 **foul +0.08** (카운트 무관).

## 5. 타자 태그 카탈로그 (`batterTagCatalog`) — 확정 10종

일반 타자: 생성 시 **0~1개**, 보스: **+1개** 가중. UI 최대 5개 안에서 mind·tendency·파생과 공존.

| id | name | 성격 | 발동 조건 | effects (초안) | 승부 기록 힌트 (예) |
|----|------|------|-----------|----------------|---------------------|
| `fast_killer` | 빠른공 킬러 | 노림형 | 투구 category `fast` + (노림 일치 **또는** targetMatch) | contact +0.06, contactQuality +8 | 빠른 공에 배트가 빨리 나옵니다. |
| `breaking_weak` | 변화구 약점 | 약점형 | 투구 `breaking`/`offspeed` + 노림 **불일치** | contact −0.08 (헛스윙↑), swing +0.04 | 변화구 타이밍을 자주 뺏깁니다. |
| `inside_weak` | 몸쪽 약함 | 코스 약점 | side `inside`, inZone | contact −0.05, timingValue −0.06 상당 | 몸쪽 공에 타이밍이 흔들립니다. |
| `inside_punish` | 몸쪽 응징 | 코스 강점 | side `inside` + 실투(의도와 다른 inside 또는 centerMistake 인접) | contactQuality +10 | 몸쪽 실투를 받아쳤습니다. |
| `outside_chase_foul` | 바깥쪽 추격 | 컨택형 | side `outside` + swung | foul +0.08 | 바깥 공을 파울로 막아냅니다. |
| `low_ball_grounder` | 낮은 공 골퍼 | 코스 강점 | height `low` + 인플레이 | contactQuality −5 but inPlay 유지 보정: weak grounder out 확률 −0.08 | 낮은 공도 땅볼로 살려냅니다. |
| `high_fast_vulnerable` | 높은 공 취약 | 코스 약점 | height `high` + category `fast` | contact −0.10, swing +0.05 | 높은 직구에 배트가 뜁니다. |
| `full_count_heart` | 풀카운트 강심장 | 승부형 | count 3-2 | !inZone chase −0.12, inZone swing +0.06 | 풀카운트에서 승부를 고릅니다. |
| `texas_luck` | 약타 생산 | 운형 | timing 「완전히 속음」 + inPlay 분기 | texasHitChance +0.08 (기본 0.10 → 0.18) | 속았는데도 약한 안타가 나옵니다. |
| `dp_risk` | 병살 위험 | 약점형 | height `low` + inPlay + 1루 주자 | doublePlayChance +0.10 | 땅볼이 병살 코스로 흘렀습니다. |

**부록 — tendency로 처리 (태그 아님)**

| id | name | 비고 |
|----|------|------|
| `offspeed_patience` | 느린공 참기 | §4.3 — tag로 넣을 경우 위 10종 → **11종** |

**권장:** v1 구현은 **10종 + tendency 2분할 + offspeed_patience 태그 1종 = 11 태그**, tendency는 5→6종.

## 6. 투수 태그 추가 (`pitcherTagCatalog`) — 5종

기존 14종(bonus 6)과 겹치지 않는 **대칭·빈 축**만. type: `bonus`, 스테이지 3택1 풀에 합류.

| id | name | 성격 | effects (초안) | 대응 타자 태그 |
|----|------|------|----------------|----------------|
| `inside_cheese` | 몸쪽 위협 | 코스 | inside 제구 +4, brush 볼 intent 시 contactQuality −3 | 몸쪽 약함 / 응징 |
| `high_fast_lift` | 고속 직구 위 | 구종 | fast + row≤0(높은 존) quality +3, whiff +0.04 | 높은 공 취약 |
| `full_count_command` | 풀카운트 제구 | 승부 | 3-2 strike intent control +5, walkPressure −2 | 풀카운트 강심장 |
| `pattern_shuffler` | 배합 교란 | 심리 | 계열 전환 시 mixing relief 추가 suspicion −2 (1구 1회) | memory v1 (섞기 보상) |
| `texas_suppress` | 약타 억제 | 대응 | 완전히 속음 texasHitChance −0.06 (상대) | 약타 생산 |

**넣지 않음 (기존과 중복)**

| 후보 | 이유 |
|------|------|
| 초구 습격 | `초구 강세`와 동일 |
| 2스트 특화 강화 | `two_strike_specialist` 존재 |
| 헛스윙/땅볼/낮은·바깥 제구 | bonus 6종에 이미 있음 |

## 7. 수치·밸런스 초안

### 7.1 타자 태그

- 단일 태그 effect는 **swing/contact/foul/chase ±0.04~0.12** 구간  
- contactQuality는 **±5~10** (장타형 slugger 보스 +16보다 낮게)  
- `texas_luck`: 기본 10% → 18% (과도하면 15%로 하향)  
- `dp_risk`: 병살 base 0.14에 +0.10 → **cap 0.32** 권장  

### 7.2 투수 태그

- `pattern_shuffler`: mixing relief −2는 **의심 cap·memory v1**과 합산 시 **1구 −6 이하**로 clamp  
- weakness 태그와 동시 보유 가능 (약점 공개 조건은 기존 유지)

### 7.3 중첩 cap (전역)

```
chasePenalty 합산 cap: −0.18
foulBonus 합산 cap: +0.22
동일 공 mind + tendency + tag + memory modifier → swing/contact 각 ±0.35 cap
```

## 8. 생성·부여 규칙

### 8.1 타자

```
generateLineup():
  mind = pickMindType()          // 기존
  tendency = pickHiddenTendency() // 6종 (초구 2분할 반영)
  memory = pickMemoryGrade()     // 기존
  tags = []
  if random < 0.55: tags.push(weightedPick(batterTagCatalog))
  if isBoss && random < 0.70: tags.push(weightedPick(bossWeightedTags))
  bossGimmick = pick(bossGimmicks) // 기존, tag와 중복 effect 금지表 참고
```

**보스 tag 가중:** `fast_killer`, `inside_punish`, `full_count_heart`, `texas_luck` ↑

**타순 가중 (예)**

| 타순 | tag 가중 |
|------|----------|
| 1~2 | full_count_heart, offspeed_patience, breaking_weak ↓ |
| 3~4 | fast_killer, inside_punish, high_fast_vulnerable |
| 5~6 | texas_luck, dp_risk ↓ |
| 7~9 | breaking_weak, low_ball_grounder, inside_weak |

### 8.2 투수

- 기존과 동일: base 1 + 스테이지 클리어 bonus 3택1  
- `generateStageTagChoices()` 풀에 §6 **5종 bonus** 추가  
- 프로필 가중 예: command → full_count_command, breaking → pattern_shuffler, power → high_fast_lift

## 9. UI·피드백

### 9.1 보여줄 것

- 타자 카드: tag name + mind hint + tendency label (최대 5)  
- 태그 클릭 → **한 줄 설명** (투수 태그와 동일 UX)  
- 발동 시 승부 기록 **짧은 힌트** (§5 표) — **수치 미노출**

### 9.2 숨길 것

- effects 수치, 중첩 cap, 내부 id  
- mind / memory / 역노림 내부 상태
