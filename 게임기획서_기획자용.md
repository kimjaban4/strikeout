# 마운드 심리전 — 기획자용 전체 설명서

**문서 목적**  
개발·아트·기획이 처음 이 프로젝트를 받았을 때, **“지금 빌드가 실제로 어떻게 돌아가는지”** 한 번에 파악할 수 있게 쓴 문서입니다.  
코드를 읽지 않아도, 이 문서만으로 플레이 루프·규칙·UI·심리전·성장 구조를 이해할 수 있습니다.

**문서 성격**  
- **현행 구현 기준** (2026년 5월, `game.js` / `js/` 분리 모듈 반영)  
- 예전 `게임기획서.md`(3이닝 단일·다크 UI)와 **다릅니다**  
- 수치·확률은 “체감 방향” 위주이며, 세부 밸런스는 코드에서 조정됩니다  

**실행 방법**  
`야구게임` 폴더에서 `node server.js` → 브라우저 `http://127.0.0.1:4173`

## 1. 30초 요약 — 꼭 먼저 읽기

| 질문 | 답 |
|------|-----|
| **누구를 조작하나?** | 투수만. 타격·주루·수비 시프트 없음. |
| **한 판 목표는?** | 3개 스테이지를 **실점 한도 안에서** 이겨 나가기. |
| **핵심 재미는?** | 타자의 **숨은 노림수**를 읽고, **반대 구종**을 던지는 심리전. |
| **반복하면?** | 같은 패턴을 읽히면 **의심 게이지**가 오르고 타자가 강해짐. |
| **잘하면?** | 삼진·병살 등으로 **스탯·구종·태그**가 성장. |
| **어떻게 던지나?** | 구종 선택(1~5키) → **5×5 히터박스** 클릭(가운데 스트, 바깥 볼). |
| **패배 조건?** | 현재 스테이지 **실점 한도 초과** (5→4→3). |
| **승리 조건?** | 스테이지3 **7이닝**을 실점 3 이하로 마침. |

**한 문장**  
“타자가 뭘 기다리는지 추리해서 반대로 던지고, 패턴이 읽히지 않게 섞으면서 3이닝→5이닝→7이닝을 실점 한도 내로 버티는 **투수 심리전 로그라이크**.”

## 2. 이 게임이 주는 경험

### 2.1 플레이어 판타지

- 나는 에이스 투수다.  
- 매 타자마다 **“지금 이 사람, 뭐 노리지?”**를 읽는다.  
- 스트라이크존 한 칸, 구종 한 개가 **승부의 선택**이다.  
- 잘 던지면 팀이 살고, 내 투수가 **경기 안에서 성장**한다.  
- 같은 실수(패턴 노출)를 반복하면 **읽혀서** 맞는다 — 그래서 매 공이 긴장된다.

### 2.2 재미의 네 기둥

1. **불완전 정보** — 노림수는 처음에 안 보인다. 단서는 스윙·타이밍·승부 기록뿐.  
2. **선택이 확률을 바꾼다** — 맞는 추리 ≠ 100% 성공이지만, 좋은 선택은 체감상 확실히 유리하다.  
3. **짧은 판 + 반복 플레이** — 스테이지 단위로 끊기고, 다시 도전하기 쉽다.  
4. **로그라이크 성장** — 같은 투수로 경기가 길어질수록 구성이 달라진다(태그·스탯·구종).

### 2.3 이 게임이 **하지 않는** 것

- 팀 운영·교체투수·타순 직접 편성  
- 타격 미니게임·주루 조작  
- 풀 9이닝 시뮬 전체  
- 리그·시즌·멀티플레이  

→ **“투수 vs 타자, 한 공씩”**에만 집중합니다.

## 3. 한 판의 큰 그림

### 3.1 메타 구조 (로그라이크 3스테이지)

```
[선발 투수 1명 고정]
    ↓
┌─ 스테이지 1 ─────────────────┐
│  3이닝 수비 · 실점 5 이하    │  ← 패배: 5실점 도달
│  상대 9명 1바퀴 + 보스 1명   │
│  클리어 → 보조 태그 3택1     │
└──────────────────────────────┘
    ↓ (실점·이닝 리셋, 새 9명)
┌─ 스테이지 2 ─────────────────┐
│  5이닝 · 실점 4 이하         │
│  타자 스탯 상승              │
│  클리어 → 보조 태그 3택1     │
└──────────────────────────────┘
    ↓
┌─ 스테이지 3 (최종) ──────────┐
│  7이닝 · 실점 3 이하         │
│  클리어 → **게임 승리**      │
└──────────────────────────────┘
```

- **한 경기 안에서** 투수 1명·구종·태그·스탯이 유지됩니다.  
- 스테이지가 바뀌면 **실점·주자·이닝·타순은 리셋**되지만, **투수 성장은 유지**됩니다.

### 3.2 시간 축 (한 타석 안)

```
타자 등장 → [구종 고민] → [히터박스 코스] → 투구
    → 결과 연출(공·배너·타이밍 문구)
    → B/S/O·주자·실점 갱신
    → 승부 기록 한 줄 추가
    → (타석 종료 시) 보상 또는 다음 타자
```

평균적으로 타석당 **여러 공**이 오가며, 2~6구 내외로 타석이 끝나는 경우가 많습니다.

## 4. 처음부터 끝까지 — 플레이어가 겪는 순서

### 4.1 타이틀 / 새 경기

- 상단 **「새 경기」** 또는 결과 화면 **재시작**.  
- BGM 토글(화면 바깥 탭).  
- 곧바로 **선발 투수 선택** 화면으로.

### 4.2 선발 투수 선택 (로그라이크 시작)

**보이는 것**

- 카드 3장: **이름**, **타입 라벨**(예: 파워 피처, 제구형 투수).  
- 초상(랜덤).

**안 보이는 것 (선택 후 공개)**

- 구속·제구·변화·멘탈·예측 수치  
- 보유 구종 3~5개  
- 기본 태그 1개  
- 숨은 약점 태그 2개  

**기획 의도**  
선발은 “느낌”만 보고 고르고, 인게임에서 정보가 열리며 **내 빌드**가 완성된다.

### 4.3 스테이지 진입

- **「STAGE 1 · 3 INNINGS」** 같은 전체 화면 오버레이.  
- 로그: 「○○ 등판. ○○ 포수와 호흡. N실점 전에 M이닝 버티기」.  
- 첫 타자가 보스면 **「보스배터 / 이름」** 배너, 아니면 **「NEXT BATTER · N번 이름」**.

### 4.4 인게임 루프 (한 타자)

| 단계 | 플레이어 행동 | 시스템 반응 |
|------|----------------|-------------|
| 1 | 타자 카드·태그·의심 게이지 확인 | 해당 타자 **노림수·성향** 내부 생성 (비공개) |
| 2 | 노림수 추천·추정 바 참고 | 투수 **예측** 스탯에 따라 추천 정확도 변동 |
| 3 | 구종 버튼 또는 1~5키 | 선택 구종 하이라이트, km/h 표시 |
| 4 | 히터박스 칸 클릭 | 스트/볼 의도 확정 → **한 공 시뮬** |
| 5 | 결과 관람 | 공 궤적, STRIKE/HIT 배너, 타이밍 문구, 승부 기록 |
| 6 | 반복 또는 타석 종료 | 볼넷·삼진·안타·병살 등 |

**타석이 끝나는 대표 경우**  
삼진, 볼넷, 인플레이 아웃, 병살, 안타·2루타·홈런, 실책 출루, (실점 한도 도달 시) **경기 종료**.

### 4.5 타석 종료 후

**A. 보상이 있는 경우** (우선순위 높음)

- 스윙 삼진 / 루킹 삼진 / 병살 / 보스 제압  
- 화면 가운데 **보상 3택1** 오버레이  
- 고르기 전까지 **투구 불가**

**B. 보상 없는 경우**

- 잠시 후 **자동으로 다음 타자** (이닝 체인지 배너가 있으면 그 후)

**C. 3아웃**

- 이닝 종료 로그·**INNING CHANGE** 배너  
- 다음 이닝 첫 타자

**D. 스테이지 이닝 수 달성**

- 스테이지 클리어 처리  
- (타석/보상 흐름 끝난 뒤) **스테이지 태그 보상** → 다음 스테이지 오버레이

### 4.6 패배 / 승리

**패배**

- 상단 실점이 **한도에 도달** (예: 스테이지1에서 5점).  
- 안타·홈런 직후에는 **결과를 잠시 본 뒤** 게임 오버 (너무 빨리 팝업 안 뜸).  
- 결과 화면: 실점, 투구 이닝, 삼진, 병살, 피안타, 볼넷, 보상 횟수, 보스 관련 수치 등.

**승리**

- 스테이지 3의 7이닝을 **실점 3 이하**로 완료.  
- 승리 메시지 + 동일 통계 요약.

## 5. 화면 구성과 정보 배치

### 5.1 톤

- **밝은 UI**, 카드형, 읽기 쉬운 숫자.  
- 경기장·캐릭터 일러스트 중심 (에셋 폴더 `assets/images` 등).  
- 정보가 많으면 **카드 접기/펼치기**로 상단(이름·사진·태그)은 고정.

### 5.2 레이아웃 (기획자 관점)

```
┌─────────────────────────────────────────────────────────────┐
│  타이틀 · 이닝/실점/목표 · B●● S●● O●● · 주자 다이아 · 새경기  │
├──────────────┬──────────────────────────┬───────────────────┤
│  [투수 카드]  │   타자 실루엣 · 타이밍    │  [타자 카드]       │
│  구종 1~5    │   5×5 히터박스           │  의심 게이지       │
│  코스 안내   │   (공 마커·판정 뱃지)     │  노림수 추정 바    │
│              │                          │  승부 기록         │
│              │                          │  노림수 추천       │
└──────────────┴──────────────────────────┴───────────────────┘
        (오버레이: 선발선택 / 보상 / 스테이지 / 결과 / 큰 판정)
```

### 5.3 플레이어에게 **항상** 보이는 정보

- 이닝, 실점, **「N실점 시 종료」** 목표  
- B/S/O (원형 점)  
- 1·2·3루 주자 점등  
- 현재 타자 이름·타순·태그(최대 6개, 2줄)  
- 투수 이름·태그·컴팩트 스탯  
- 선택 구종·km/h  
- 의심 게이지 %  
- 노림수 추정(3계열 바)  
- 노림수 **추천** 카드(확신도 %)  
- 승부 기록(최근 공들)

### 5.4 플레이어에게 **안 보이는** 정보 (추리 대상)

- 이번 타석 **진짜 노림수** (강속구/변화구/느린공 계열)  
- 타자 **숨은 성향** ID (초구형·안구형 등 — 태그로 힌트만)  
- **역노림** 여부  
- **거짓 단서**가 섞였는지  
- 다음 스테이지 보스 **몇 번 타자**인지 (매 스테이지 랜덤)

### 5.5 화면이 “잠기는” 때 (입력 불가)

| 상황 | 이유 |
|------|------|
| 선발 선택 중 | 아직 경기 미시작 |
| 보상 선택 중 | 로그라이크 선택 강제 |
| 타석 종료 직후 잠깐 | 연출·자동 진행 |
| 경기 종료 대기 | 홈런 등 큰 결과 후 지연 |
| 게임 오버 | 종료 |

## 6. 야구 규칙 — 이 게임에서 실제로 적용되는 것

### 6.1 카운트

- **4볼 → 볼넷** (1루, 만루 시 득점)  
- **3스트라이크 → 삼진** (루킹 / 헛스윙 구분 → 보상 차등)  
- **파울** — 스트라이크 2개 미만일 때만 S+1 (2스트 후 파울은 S 유지)

### 6.2 주자·득점

- 안타·볼넷·실책·홈런 시 주자 진루·득점  
- **병살**: 1루 주자 + 아웃 2개 미만 + 낮은 타구 품질 등 조건 시 발생  
- 홈런: 주자 전원 + 타자 득점

### 6.3 아웃·이닝

- 3아웃 → 이닝 종료, 주자 클리어, 이닝+1  
- 현재 스테이지 **목표 이닝**을 채우면 스테이지 클리어  
  - 예: 스테이지1은 3이닝 **끝낸 뒤**에도 실점 5 미만이어야 함

### 6.4 특수 결과

| 결과 | 플레이어 체감 | 비고 |
|------|----------------|------|
| TEXAS HIT! | 속였는데 약타 | 완전 속임 + 낮은 컨택 품질 + 소확률 |
| DOUBLE PLAY! | 1루 주자 정리 | 변화구·낮은 코스와 궁합 |
| ERROR! | 수비 실책 | 약타구에서도 가끔 출루 |
| WALK! | 볼넷 | 2볼넷 이상 시 약점「볼넷 위험」공개 가능 |

## 7. 투구 조작 — 구종과 히터박스

### 7.1 구종 선택

- 투수마다 **3~5개** 구종 (프로필·랜덤).  
- UI: 아이콘 + 이름 + **km/h** + 한 줄 설명.  
- 키보드 **1~5**: 보유 순서대로만 (4개면 5번 무효).

**구종 풀 (게임 전체)**

| 이름 | 계열 | 역할 한 줄 |
|------|------|------------|
| 포심 | 강속구 | 정면 승부 |
| 투심 | 강속구 | 땅볼 유도 |
| 싱커 | 강속구 | 낮은 땅볼 |
| 커터 | 강속구 | 빗맞힘 |
| 슬라이더 | 변화구 | 헛스윙·유인 |
| 커브 | 변화구 | 타이밍 교란 |
| 체인지업 | 느린공 | 직구 뒤 완급 |
| 스플리터 | 느린공 | 낙차 큰 승부구 |

**필수 조합 규칙**

- 강속구 계열 **최소 1**  
- 변화구 또는 느린공 계열 **최소 1**  
- 파워 피처는 강속구 비중↑, 변화구 장인은 4~5종·변화 위주

### 7.2 히터박스 (코스 + 스트/볼 의도)

**5×5 격자**

```
[ 볼 ][ 볼 ][ 볼 ][ 볼 ][ 볼 ]
[ 볼 ][ S ][ S ][ S ][ 볼 ]
[ 볼 ][ S ][ S ][ S ][ 볼 ]
[ 볼 ][ S ][ S ][ S ][ 볼 ]
[ 볼 ][ 볼 ][ 볼 ][ 볼 ][ 볼 ]
```

- **가운데 3×3**: 스트라이크 **의도**  
- **바깥**: 볼 **의도** (타자도 상황에 따라 스윙할 수 있음 — “절대 안 침” 아님)

**플레이어가 찍는 것 vs 실제 공**

| 플레이어 | 게임 |
|----------|------|
| 클릭한 칸 | **의도** (어디에 넣으려 했는지) |
| 제구·멘탈·상황 | **실제 도착 위치** (흔들릴 수 있음) |
| 화면 마커 | **실제 위치만** 표시 (의도 위치 X) |

**연속 볼 보정**  
스트라이크를 넣으려 해도 자꾸 볼이 나올 때, **연속 볼 2개 이상**이면 다음 스트라이크 의도에 존 복구 확률이 소폭 올라감 (억지 스트라이크가 아니라 ‘감각 보정’).

**볼 유도 시 내부 성격** (플레이어에게 버튼으로 안 뜸)

- 낚시 볼 / 보여주기 볼 / 버리는 볼 / 위협구 — 클릭 위치·상황에 따라 자동 분류되어 타자 반응에 영향.

## 8. 한 공이 처리되는 방식

기획자가 “왜 이 공이 스트라이크가 됐지?”를 이해할 때 보는 순서입니다.

### 8.1 제구 (위치 결정)

입력: 구종, 의도(스트/볼), 클릭 칸, 투수 제구·멘탈, 주자 압박, 태그 보정.  
출력: **실제 존 안/밖**, 코스 라벨(예: 「낮은 바깥쪽」「정중앙」).

- 스트라이크 의도인데 **제구 실패** → 볼 가능  
- 볼 의도인데 **제구가 몰림** → 스트라이크 가능  
- **의도와 다른 곳**에 공이 가면 → 중앙 실투·장타 위험 등 별도 보정

### 8.2 스윙 여부

- 존 안/밖 기본 스윙률 다름  
- **노림 일치**면 스윙↑, 불일치면↓  
- 카운트(2-0, 3-2, 2스트 등), 타자 접근(적극/신중/초구/보호), 구종 특성, **의심·패턴 노출**, 볼 플랜

**스윙 안 함**

- 존 안 → **루킹 스트라이크**  
- 존 밖 → **볼**

### 8.3 스윙함 → 타이밍

- 0~1 수치 → **완벽 / 좋음 / 불안정 / 늦음 / 너무 빠름 / 밀림 / 완전히 속음** 등 UI 문구  
- 노림·구속·구위·반복 구종·속임 보정 반영

### 8.4 컨택

- 실패 → **헛스윙 삼진**  
- 성공 + 타이밍 애매 → **파울** (2스트 전만 S 증가)  
- 성공 + 인플레이 → 타구 품질 점수

### 8.5 인플레이 결과 (타구 품질 순)

대략적인 우선순위:

1. **완전히 속음** + 일부 조건 → TEXAS 안타 또는 땅볼 아웃  
2. 품질 매우 높음 → **홈런**  
3. 높음 → **2루타**  
4. 중간 → **1루타**  
5. 1루 주자 + 낮은 품질 → **병살** 가능  
6. 약타구 → **실책** 소확률  
7. 그 외 → **땅볼/뜬공 아웃**

### 8.6 승부 기록에 남는 것 (플레이어 읽는 텍스트)

예시 형태:

- `투심 147km/h`  
- `이번 타자 3구 · 낮은 바깥쪽 · 스트라이크 승부`  
- `스윙 늦음 · 배트가 공 밑을 지나갔습니다.`  
- 타석 끝나면 **노림수 공개** 문구가 붙을 수 있음  

번호 코스(「3번 존」)는 **쓰지 않음** — 자연어 코스만.

## 9. 심리전 — 이 게임의 본체

### 9.1 노림수 (Hidden Target)

**매 타석 타자 1명당 1개 (계열)**

| 노림수 | 의미 | 플레이어가 던질 때 |
|--------|------|-------------------|
| 강속구 계열 | 빠른 공 기대 | 다른 계열 → 헛스윙·지켜봄 단서 / 같은 계열 → 장타 위험 |
| 변화구 계열 | 슬라이더·커브류 | 강속구로 속이기 좋음 |
| 느린공 계열 | 체인지·스플리터류 | 강속구로 앞타이밍 유도 |

타순 아키타입(출루형·중심형·장타형·하위형)이 **노림 가중치**에 영향.  
**직전 타석 결과**도 다음 타자 노림에 반영 (예: 변화구에 삼진 당했으면 변화구 노림↑).

### 9.2 노림수 추정 UI (Read)

- fast / breaking / offspeed **세 막대 점수**  
- 매 공 결과마다 갱신 (스윙·타이밍·파울·볼 반응 등)  
- **거짓 단서** 발생 시 잘못된 계열 점수가 올라감 → 교활한 타자 주의

### 9.3 노림수 추천 (Recommendation)

- 투수 스탯 **「예측」**이 높을수록 **진짜 노림**을 맞출 확률↑  
- 맞추면 **-counter 구종** 추천 (상대 계열을 깨는 구종)  
- **지금 던지는 투수가 가진 구종만** 추천 (없는 구종 이름 안 나옴)  
- 확신도 % 표시  
- 교활형 + 의심 70+ → 추천 신뢰도 하락

### 9.4 의심 게이지 (Suspicion)

**“타자가 나를 얼마나 읽었나”** 0~100%

**올라가는 행동 (패턴 신호 예)**

- 같은 구종 또 던짐  
- 같은 계열 반복  
- 같은 코스·같은 높이 반복  
- 같은 스트/볼 의도 반복  
- 같은 카운트에서 같은 선택 반복  
- 주자 상황 같을 때 같은 구종  
- 배합 순서가 노출됨 (예: 직구 뒤 변화 패턴)

**내려가는 행동**

- 구종·계열·코스·의도를 섞음

**임계값 체감**

| 의심 | 효과 |
|------|------|
| 58+ & 신호 있음 | 「타자가 ○○을 의식합니다」— 패턴 노출 |
| 72+ & 교활/적응/도박 | **역노림** — 표면 반응 믿기 위험 |
| 높을수록 | 거짓 단서 확률↑, 컨택·파울 보정↑ |

### 9.5 타자 마음 (Mind) — 반응 신뢰도

| 유형 | 플레이어에게 보이는 힌트 | 기획 |
|------|-------------------------|------|
| 정직형 | 정직스윙 | 스윙·로그 비교적 믿을 만함 |
| 교활형 | 포커형 | 속이는 반응·거짓 단서 |
| 적응형 | 패턴형 | 패턴 학습 빠름 |
| 도박형 | 한방형 | 한 방 노림 |

### 9.6 숨은 성향 (Tendency) — 태그로 힌트

초구형, 인내형, 안구형, 응징형, 즉응형 등 — 특정 카운트·코스에서 스윙/파울/추격 변화.

### 9.7 기억력 (Memory)

낮음 / 보통 / 높음 / 천재형 — 직전 몇 구를 얼마나 기억하는지. 천재형·보스(천재타자 기믹)는 **전역 패턴**까지 참고.

### 9.8 포수·카운트 카드 (분위기 텍스트)

- 포수 타입(안정/공격/분석/배짱)에 따른 **짧은 조언 문구**  
- B/S/O 조합별 **심리 카드** — 승부 분위기 연출용 (숫자 로직 보조)

## 10. 투수 — 생성·능력·구종·태그

### 10.1 프로필 4종

| 프로필 | 느낌 | 구종 |
|--------|------|------|
| 파워 피처 | 강속구 위주 | 강속 2 + 변화/느린 1 |
| 변화구 장인 | 많은 구종 | 4~5종, 슬라이더·커브 |
| 제구형 | 안정 | 3~4종 |
| 균형형 | 올라운드 | 3~4종 랜덤 |

### 10.2 능력치 5종 (선발 시 랜덤 범위, 보상으로 성장)

| 스탯 | 플레이어 체감 |
|------|----------------|
| 구속 | 빠른 공 위력·표시 km/h |
| 제구 | 스트라이크 넣기·볼넷 줄이기 |
| 변화 | 변화구·느린공 위력 |
| 멘탈 | 주자·위기 시 흔들림 감소 |
| 예측 | **노림수 추천** 정확도 |

### 10.3 태그 3층

**① 기본 태그** — 프로필에 따라 1개, 선발 후 공개  
**② 보조 태그** — 스테이지 클리어 시 3택1, 중복 없음  
**③ 약점 태그** — 2개 숨김, 조건 충족 시 이름·설명 공개

약점 공개 예:

- 홈런 허용 → 장타 위험  
- 볼넷 2회 → 볼넷 위험  
- 풀카운트 3회 → 풀카운트 불안  
- 연속 출루 2회 → 위기 흔들림  

보조 태그 예: 2스트 특화, 낮은 코스 장인, 헛스윙 유도, 땅볼 유도, 멘탈 회복, 초구 강세 …

## 11. 타자 — 라인업·보스·성향

### 11.1 9명 라인업

- 매 스테이지 **새 9명** (이름·얼굴 랜덤).  
- 타순 1~9: 아키타입으로 스탯·노림 가중치 다름.  
- **보스 1명**: 1~9번 중 랜덤 슬롯, 스탯↑, 전용 기믹.

### 11.2 스테이지가 올라가면

- 타자 스탯 **단계적 상승** (스테이지2·3에서 더 큼).  
- 의심 초기값에 스테이지 보너스.

### 11.3 보스 기믹

| 보스 | 플레이어가 느낄 위험 |
|------|---------------------|

| 천재타자 | 패턴·역노림 극강 |
| 눈야구 | 볼 유도 잘 참음 |

### 11.4 타자 카드 UI

## 12. 보상과 성장 — 로그라이크 루프

### 12.1 보상이 뜨는 때

| 트리거 | 보상 크기 |
|--------|-----------|
| 스윙 삼진 | 작음 |
| 루킹 삼진 | 중간 |
| 병살 | **큼** (+추가 선택지) |
| 보스 제압 | 보스 보너스 풀 |
| (스테이지 클리어) | **보조 태그** 3택1 (별도) |

**연출 순서**  
큰 판정(삼진·병살 등) → 잠시 대기 → 보상창 → (있으면) 다음 타자 / 이닝 / 스테이지.

### 12.2 일반 보상 3택1 예

- 제구/구속/변화/멘탈/예측 **+1~2**  
- 방금 쓴 구종 **제구 +3~5**  
- 슬롯 여유 시 **새 구종 1개** 추가  

### 12.3 스테이지 보상

- **보조 태그** 하나 영구 추가 (프로필에 맞는 태그 가중).

### 12.4 한 경기 안 성장 곡선

```
선발(기본태그+약점 숨김)
  → 타석마다 작은 보상으로 스탯·구종 강화
  → 스테이지마다 태그 추가
  → 후반 스테이지·보스에서 빌드 검증
```

## 13. 연출·피드백·텍스트 톤

### 13.1 공·판정

- 투구 시 **공 궤적 애니메이션** (구종별 궤적 차이는 있으나 과장도는 제한적)  
- 히터박스 위 **실제 도착 마커** + 스트/볼 뱃지  
- 큰 결과: **STRIKE OUT!**, **HOME RUN!**, **DOUBLE PLAY!** 등 중앙 배너  
- 스트/볼/파울은 존 근처 작은 피드백  

### 13.2 타이밍 배지

- 투구 **후**에만 짧게 (「완벽」「늦음」「완전히 속음」 등)  
- 투구 **전**에는 미리 안 알려줌 → 추리 유지

### 13.3 BGM·효과음

- BGM 루프, 토글  
- 안타 / 홈런 / 헛스윙 등 효과음  

### 13.4 문체

- 짧고 명확, **한 줄 코멘트** 위주  
- 승부 기록은 **해설처럼** 읽히게 (「간신히 걷어냈습니다」 등)  
- 패턴 노출 시 **「타자가 같은 구종 반복을 의식하기 시작합니다」** 류

## 14. 난이도·페이싱·패배 체감

### 14.1 난이도 곡선

| 구간 | 압박 |
|------|------|
| 스테이지1 3이닝 | 실점 여유 5 — 학습 구간 |
| 스테이지2 5이닝 | 실점 4, 타자 강화 |
| 스테이지3 7이닝 | 실점 3, 장기전·실수 누적 |

### 14.2 플레이어가 지는 대표 이유

1. **노림을 잘못 읽고** 맞는 구종을 던짐 → 장타  
2. **패턴 반복** → 의심↑ → 역노림·거짓 단서  
3. **제구 실패** → 볼넷·실투 코스  
4. **실점 한도**가 스테이지마다 빡빡해짐  

### 14.3 밸런스 철학 (의도)

- 정답 게임이 **아님** — 좋은 선택이어도 가끔 홈런  
- 나쁜 선택은 **납득 가능한 이유**로 실점 (로그·단서로 설명)  
- **다음 공에서 바꿀 수 있다**는 희망 (카운트·구종·코스)

## 15. 플레이 예시 시나리오 3개

### 시나리오 A — 정직한 3번 타자, 변화구 노림

1. 타자 태그: 「중심형」「정직스윙」. 추정 바가 **변화구** 쪽↑.  
2. 추천: 「포심으로 타이밍 압박」.  
3. 플레이어: **포심**, 가운데 스트라이크 → 헛스윙 or 약파울, 추정 바 **변화구** 더↑.  
4. 2구: **슬라이더** 바깥 낮은 볼 유도 → 스윙 → 삼진.  
5. **스윙 삼진 보상** → 예측 +2 선택.  

**기획 포인트**: 정직형은 로그가 비교적 믿을 만함.

### 시나리오 B — 패턴 노출 후 역노림

1. 2스트에서 **슬라이더** 바깥, 또 **슬라이더** → 의심 60+, 「같은 구종 반복」.  
2. 3구: **체인지업** 던짐 (느린공으로 속이려 함).  
3. 교활형 타자 + 역노림 → 실제로는 **강속구** 노림이었을 수 있음 → **정타** → 1루타.  
4. 승부 기록에 「타자가 배합 순서를 의식합니다」 류.  

**기획 포인트**: 반복은 처벌, 섞어야 함.

### 시나리오 C — 스테이지 클리어 직전

1. 3이닝 2아웃, 실점 4 (한도 5).  
2. 보스 5번 — 「홈런왕」. 가운데 맞으면 끝장.  
3. 코너 바깥 낮은 존 + 변화구로 땅볼 → **병살** (1루 주자 있었으면).  
4. **병살 보상** (큼) + 이닝 종료 → 스테이지 클리어.  
5. **보조 태그 3택1** → 「땅볼 유도」 선택.  
6. **STAGE 2 · 5 INNINGS**, 실점 0으로 리셋, 새 9명.
