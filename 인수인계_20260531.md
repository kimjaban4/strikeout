마운드 심리전(`야구게임/`) 작업 인수인계. 이번 세션에서 진행한 6개 작업의 변경 사항·이유·파일 위치를 정리합니다.

## 0. 작업 워크플로 (필독)

- **수정 기준 소스는 `game.js`** (모놀리식). 직접 편집은 여기서.
- 편집 후 **반드시** 모듈 재생성: `node tools/split-game-modules.mjs`
  - → `js/00-constants.js`, `js/01-state.js`, `js/02-game-core.js` 가 새로 만들어짐
  - `js/03-pitch-progression.js`는 split 대상 아님 (직접 관리)
- 실행: `npm start` → `http://127.0.0.1:4173`
- 게임은 `MountPsycho` 네임스페이스(IIFE). classic script.
- CSS 변경 시 `index.html`의 `styles.css?v=...` 캐시 버전 갱신 + 강력 새로고침(Ctrl+Shift+R).
- UI는 `.card-v2` 스코프로만 작업. 다른 UI 영향 최소화.

## 1. 투수 태그 설명 — 클릭한 섹션 바로 아래 인라인 표시

**무엇:** 투수 카드에서 보조/약점/핵심 태그 칩을 클릭하면, 설명이 **그 태그가 속한 섹션 바로 아래**에 펼쳐지도록 변경. (이전엔 카드 하단 고정 위치)

**구현:**
- `renderPitcherTags()` — 각 섹션(`data-tag-section="support|weakness"`)에 `<div class="pitcher-tag-detail-slot">` 추가
- `showPitcherTagDetail(tagId, anchorButton)` — 클릭한 버튼이 속한 섹션 슬롯으로 설명 박스(`#pitcherTagDetail`)를 이동시켜 표시. 핵심태그(이름 옆 칩) 클릭 시엔 `#pitcherName` 아래로.
- `parkPitcherTagDetail()` / `repositionOpenPitcherTagDetail()` — render 때 위치 보존
- 이벤트: `els.pitcherTags`, `els.pitcherName` click → `showPitcherTagDetail(id, button)`

**파일:** `game.js`(renderPitcherTags ~3280, showPitcherTagDetail 등), `styles.css`(`.pitcher-tag-detail-slot`, `.pitcher-tag-detail` 인라인 스타일 ~5535), `index.html`(`#pitcherTagDetail`)

**참고:** card-v2 그리드에서 `tagdetail` 영역행은 제거됨(이제 섹션 내부 flow).

## 2. 테스트 콘솔(`test.html`) 연결 문제 해결

**증상:** 테스트 콘솔이 "게임 스크립트 로딩…"에서 멈추고 컨트롤이 전혀 안 먹음.

**원인 2가지:**
1. **`file://`로 열면** iframe 보안으로 부모→자식 접근 차단 → 직접 브리지 실패.
2. **`pitchBurdenConfig` / `pitchMasteryConfig` / `pitchLevelNames` 상수가 누락**되어 있었음. `js/03-pitch-progression.js`가 이 상수에 의존 → 연결 후 `tiers` 참조 오류.

**해결:**
- **게임 측 postMessage 브리지** 추가: `game.js`의 `setupTestConsoleBridge()` (채널 `"mount-psycho-test"`). `ping` / `exec` / `snapshot` 명령 처리. iframe 보안으로 직접 접근 막혀도 RPC로 동작.
- **`test.html` 재작성**: `bridgeMode`("direct" | "rpc") 자동 감지, `gameExec()`/`pullSnapshot()`로 상태 조작·조회. `file://` 경고 배너 추가.
- **상수 복구**: `game.js`와 `js/00-constants.js`에 `pitchBurdenConfig` 등 3종 다시 추가.

**파일:** `game.js`(setupTestConsoleBridge, 상수), `test.html`(전체 스크립트), `js/00-constants.js`

**테스트 방법:** 반드시 `npm start` 후 `http://127.0.0.1:4173/test.html`. 제목 옆 점 초록 + "연결됨". 투수 없으면 **랜덤 새 투수** 먼저 클릭.

## 4. 핵심태그 설명 전면 재작성 (B안)

**무엇:** 핵심태그 12종 설명이 추상적("압박합니다")이라, **"언제 / 무엇이 / 얼마나"** 가 드러나게 재작성.

**파일:** `game.js` `coreTagCatalog`(~375)의 각 `description`.

**예:** 하이볼러 → "높은 코스에 강속구를 던지면 구위가 오르고 헛스윙 유도가 쉬워집니다. 강속구 전반의 위력도 조금 강해집니다."

**UI 노출 방식:** 칩은 이름만, 설명은 클릭 시 섹션 아래 패널에서 자동 줄바꿈(2~3줄). 그래서 긴 설명 OK. (1번 참고)

**미완:** 강화 카드(`coreUpgradeTemplatesByFamily`의 suffix들)에는 아직 설명 없음. 보상창의 "핵심 방향 심화"는 카테고리 라벨일 뿐. 추후 suffix별 한 줄 설명 추가 검토 가능.
