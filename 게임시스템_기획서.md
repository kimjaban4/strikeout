## 1. 시스템 한눈에 보기

### 1.1 한 투구의 처리 파이프라인

```
[플레이어 입력]
  구종 ID + 히터박스 칸(존·행·열·스트/볼 의도)
        ↓
[패턴 분석] analyzePitcherPattern()
  → 의심 게이지 Δ, 패턴 신호, 역노림, 거짓 단서, 볼 플랜
        ↓
[제구] resolvePitchLocation()
  → 실제 행/열, 존 안/밖, 실투 여부
        ↓
[판정] resolvePitch()
  → 스윙? → 타이밍 → 컨택? → 파울? → 인플레이 품질 → 최종 결과
        ↓
[후처리]
  updateRead() · applyPitchResult() · recordPitchPattern()
```

### 1.2 핵심 설계 원칙

| 원칙 | 의미 |
|------|------|
| **선택 → 확률** | 맞는 추리가 100% 성공이 아님. 좋은 선택은 확률을 크게, 나쁜 선택은 장타·볼넷 위험 |
| **반복 → 처벌** | 같은 구종·계열·코스·의도 반복 시 의심↑ → 타자 유리 |
| **정보 비대칭** | 노림수·성향·역노림은 숨김. 승부 기록·추정 바·의심 게이지로 추리 |
| **성장은 결과 연동** | 삼진·병살·보스 제압 → 보상. 스테이지 클리어 → 태그 |

## 2. 투구 선택 구조

플레이어가 고르는 것은 **구종 1개 + 히터박스 1칸(코스 + 스트/볼 의도)** 입니다.  
이 두 입력이 카운트·타자 성향·심리전 수치와 맞물려 **제구 → 스윙 → 결과**로 이어집니다.

### 2.1 입력 레이어

| 입력 | 값 | 역할 |
|------|-----|------|
| 구종 | `pitch.id` (포심·슬라이더 등 8종 풀 중 3~5개 보유) | 계열(fast/breaking/offspeed), 구위, 구종 특수 효과 |
| 존 | 1~9 (5×5 UI의 9개 클릭 영역) | 의도 코스(높이·좌우) |
| 의도 | `strike` / `ball` | 가운데 3×3 = 스트, 바깥 = 볼 |
| 행·열 | -1~3 (볼 유도 시 세부 위치) | 볼 플랜 자동 분류, 실제 도착 계산 |

### 2.2 구종 계열과 노림수 관계

타자 **노림수**는 `fast` | `breaking` | `offspeed` 중 1개 (숨김).

**노림 일치 판정** (`isTargetedPitchCategory`):

- 같은 계열이면 일치 (예: 노림 `breaking`, 투구 `slider` → 일치)
- `fast` vs `breaking`/`offspeed`는 **family** 기준으로 구분 (`fast` / `secondary`)

| 노림 일치 | 스윙 | 타이밍 | 컨택 | 장타 위험 |
|-----------|------|--------|------|-----------|
| **일치** | +0.20 | +0.20 (base) | +0.09 | contactQuality +14 |
| **불일치** | −0.09 | −0.11 (base) | −0.08 | contactQuality −10, 속임 +0.16 |

→ **반대 계열**을 던지면 헛스윙·루킹 단서가 나오지만, 가끔 **텍사스 안타**(10%) 가능.

### 2.3 코스(히터박스)와 제구

**의도 위치** (`intendedCourse`)와 **실제 위치** (`resolvePitchLocation`)는 분리됩니다.

```
commandScore = clamp(
  (pitch.control + 투수.제구) / 2
  − 주자압박(멘탈 기반)
  − pitch.cost × 0.75
  + 태그 제구 보너스,
  5, 98
)

effectiveCommand = commandScore + (스트 의도 && 연속볼≥2 ? min(연속볼×9, 24) : 0)

driftChance = clamp(0.68 − effectiveCommand/116, 0.025, 0.76)
→ drift 시 row/col ±1 랜덤

intentMissChance:
  볼 의도: clamp(0.42 − effectiveCommand/145, 0.04, 0.36) → 존 안으로 몰림
  스트 의도: clamp(0.36 − effectiveCommand/145 − 연속볼×0.035, 0.015, 0.28) → 존 밖으로 빗나감
```

**코스가 결과에 미치는 영향**

| 코스 조건 | 효과 |
|-----------|------|
| `inZone` (row 0~2, col 0~2) | 스윙 기본 46% |
| 존 밖 | 스윙 기본 16%, 선구 스탯으로 감소 |
| `centerMistake` (정중앙) | 스윙 +0.11~0.18, 컨택 +0.07~0.12, contactQuality +11~18 |
| `row ≥ 2` (낮은 코스) | 땅볼·병살·투심/스플리터 보너스 |
| `col ≤ 0 or ≥ 2` (바깥) | 슬라이더·커터 유인 |

### 2.4 카운트와 스윙 성향

**기본 스윙률** 위에 `countSwingAdjustment(inZone)` 가산:

| 카운트 | 존 안 | 존 밖 |
|--------|------|------|
| 0-0 | +0.04 | — |
| 2-0, 3-1류 (balls≥2, S=0) | +0.14 | −0.10 |
| 3볼 | +0.22 | −0.16 |
| 2스트 | +0.13 | +0.19 (체이스↑) |
| S > B & S<2 | +0.08 | +0.04 |
| B > S & B<3 | +0.08 | −0.05 |

**타자 접근(approach)** — 타석 시작 시 성향·랜덤으로 결정:

| approach | 스윙 보정 | 용도 |
|----------|-----------|------|
| 적극 | +0.13 | 언제든 치려 함 |
| 신중 | −0.12 | 볼 기다림 |
| 초구 | 0-0에서 +0.22 | 1구 공격 |
| 보호 | 2스트에서 +0.24 | 파울·생존 |
| 균형 | 0 | 기본 |

### 2.5 구종 × 코스 × 카운트 × 성향 — 상호작용 매트릭스

아래는 **추가 보정**이 겹치는 대표 조합입니다. (기본 공식은 §5 참고)

| 조합 | 시스템 반응 | 기획 의도 |
|------|-------------|-----------|
| 포심 + 2스트 + inZone | `pitchSpecialEffect`: quality +6, swing +0.05 | 결정구 위력 |
| 슬라이더 + 바깥 + 볼 의도(fishing) | swing +0.08, chase +0.10, contact −0.05 | 헛스윙 유인 |
| 체인지 + 직전 구종 fast | swing +0.05, contact −0.10, CQ −9 | 완급차 속임 |
| 2-0 + walkHunter 성향 + 존 밖 | chase −0.10, 선구/230 감소 | 볼넷 유도 성공 |
| 2스트 + twoStrike 성향 | foul +0.10 | 파울 늘리기 |
| 0-0 + firstPitch 성향 | swing +0.09 | 초구 공격 |
| 3-2 풀카운트 + full_count_wobble 약점 | commandScore −5 | 긴장·실투 |
| 주자 + clutch 보스 | contact +0.08, CQ +9 | 클러치 강화 |

### 2.6 볼 유도 내부 플랜 (자동 분류)

플레이어에게 버튼은 없고, **볼 의도 + 클릭 위치**로 `classifyBallIntent`가 결정:

| ID | 분류 조건 | 타자 반응 |
|----|-----------|-----------|
| fishing | row > 2 | 스윙↑, chase↑ |
| show | row < 0 | 다음 공 quality↑ (체감 변화) |
| waste | col > 2 등 | read 정보 +0.22 (추정 바) |
| brush | col < 0 | 스윙↓, CQ −4 (몸쪽 의식) |

## 3. 타자 AI 구조

타자 AI는 **한 타석 Plan** + **기억·패턴 학습** + **마음·성향·보스 기믹**으로 구성됩니다.

### 3.1 타석 Plan 생성 (`createPlan`)

타자 등장 시 1회 생성:

```
target     = weightedCategory(adjustWeightsByMemory(weights, lastAtBatMemory))
approach   = tendency.approach || random(균형/적극/신중/초구/보호)
suspicion  = clamp(보스+10 + memory.suspicion + stageIndex×4, 0, 100)
readScores = { fast: 1, breaking: 1, offspeed: 1 }
```

**노림수 가중치 (weights)** — 타순 아키타입 기본값 + 스테이지:

| 타순 | 유형 | fast | breaking | offspeed |
|------|------|------|----------|----------|
| 1~2 | 출루형 | 3 | 2 | 2 |
| 3~4 | 중심형 | 4 | 3 | 2 |
| 5~6 | 장타형 | 3 | 3 | 2 |
| 7~9 | 하위형 | 2 | 2 | 2 |

스테이지마다 전 계열 +2 (보스는 추가 +2).

### 3.2 직전 타석 기억 (`lastAtBatMemory`)

타석 종료 시 저장 → **다음 타자** 노림 가중치에 반영:

| 직전 결과 | 가중치 변화 | 해석 |
|-----------|-------------|------|
| 스윙 삼진 (swingingStrikeout) | 해당 계열 +2.2 | 「또 던져」 |
| 루킹 삼진 (lookingStrikeout) | 해당 계열 +1.5, fast +0.6 | 「속도/존 의식」 |
| 병살 (doublePlay) | 해당 계열 −1.0 (min 0.6), fast +1.0 | 「변화구 경계, 직구 기대」 |
| 안타 (hit) | 해당 계열 +1.2 | 「맞았던 공 다시 기대」 |

→ **라인업 전체가 이전 타석 결과를 학습**합니다. 연속 삼진 후 다음 타자가 같은 계열을 더 노릴 수 있음.

### 3.3 구간 기억 (`rememberedPitchesFor`)

패턴 분석에 쓰이는 **투구 기록 span**:

| 기억 등급 | span | 의심 초기 | 기억 범위 |
|-----------|------|-----------|-----------|
| 낮음 | 1 | −5 | 이번 타석 최근 1구 |
| 보통 | 3 | 0 | 이번 타석 최근 3구 |
| 높음 | 8 | +8 | 이번 타석 최근 8구 |
| 천재형 | 18 | +16 | **전역 patternMemory 18구** + 이번 타석 전체 |

천재형·천재타자 보스: `geniusEye` 기믹 → span 18, bossBonus +14.

**저장 항목** (`recordPitchPattern`):

```
pitchId, category, family, zone, side, height, intent,
count, runners, result
```

전역 `patternMemory`는 최대 40구 FIFO.

### 3.4 타자 마음 유형 (Mind)

| ID | 라벨 | reliability | fakeRate | patternLearn | AI 행동 |
|----|------|-------------|----------|--------------|---------|
| honest | 정직형 | 0.86 | 0.04 | 0.92 | 승부 기록·반응 신뢰 가능 |
| tricky | 교활형 | 0.42 | 0.20 | 1.08 | 거짓 단서·역노림↑ |
| adaptive | 적응형 | 0.62 | 0.10 | **1.26** | 패턴 학습 가장 빠름 |
| gambler | 도박형 | 0.54 | 0.16 | 0.82 | 한 방, 역노림 대상 |

### 3.5 숨은 성향 (Tendency)

| ID | approach | 전투 보정 |
|----|----------|-----------|
| firstPitch | 초구 | 0-0 swing +0.09 |
| twoStrike | 보호 | 2스트 foul +0.10 |
| walkHunter | 신중 | 존 밖 chase −0.10 |
| slugger | 적극 | centerMistake CQ +8 |
| reactive | 균형 | suspicion≥50 → contact +0.08, pattern +0.12 |

### 3.6 타자 AI 의사결정 요약

```
[타석 시작]
  노림수(target) 확정 ← weights + 직전 타석 memory
  approach 확정 ← tendency 또는 랜덤
  suspicion 초기값 ← memory + 보스 + 스테이지

[매 투구]
  1. rememberedPitches와 현재 선택 비교 → suspicion Δ
  2. targetMatch = (투구 계열 vs target)
  3. swingProbability = f(존, 카운트, approach, targetMatch, 의심, 볼플랜, 구종특수, 태그)
  4. 스윙 시 timingValue = f(예측, targetMatch, quality, repeatedPenalty, deception)
  5. contact / foul / inPlay 분기
  6. updateRead → 플레이어 추정 바 갱신 (거짓 단서 포함)
```

**역노림** (`reverseRead`): suspicion ≥ 72 **且** tricky/adaptive/gambler  
→ 표면 반응(추정 바·스윙)을 믿기 위험. 실제 target과 다른 방향으로 유리해질 수 있음.

## 4. 심리전 핵심 수치

### 4.1 의심 게이지 (Suspicion) — 0~100%

**“타자가 투수 패턴을 얼마나 읽었는가”**

#### 증가·감소 (매 투구 `analyzePitcherPattern`)

| 신호 | Δ | 비고 |
|------|---|------|
| 같은 구종 반복 | +12 | |
| 다른 계열로 전환 | −5 | |
| 같은 family 반복 | +7 | fast vs fast, secondary vs secondary |
| 같은 코스(side+height) 반복 | +10 | |
| 다른 코스 | −4 | |
| 같은 의도(스트/볼) 반복 | +5 (볼 +8) | |
| 다른 의도 | −4 | |
| 같은 카운트에서 같은 category | +8 | |
| 같은 주자 상황 + 같은 pitchId | +7 | |
| 배합 순서 노출 (prev.category=curr && last.family≠curr.family) | +8 | |

```
suspicionDelta = round((Σdelta + tendency.pattern×100 + bossBonus) × mind.patternLearn)
suspicion = clamp(이전 + delta, 0, 100)
```

- `tendency.pattern`: 즉응형 0.12 → +12 per signal batch
- `bossBonus`: 천재타자 +14

#### 임계값과 게임플레이 효과

| 구간 | 플래그 | 효과 |
|------|--------|------|
| ≥ 58 & signals ≥ 1 | **exposed** (패턴 노출) | swing +0.05~0.15, contact +0.04~0.12, foul +0.06~0.14, CQ + suspicion×10% |
| ≥ 72 & tricky/adaptive/gambler | **reverseRead** (역노림) | contact +0.06, CQ +8 |
| ≥ 70 & tricky | 추천 신뢰 −12%p | |
| ≥ 80 (UI) | 힌트 | 「역노림 위험」 |

#### 감소 방법 (플레이어)

- 구종·계열·코스·스트/볼 의도를 **섞기**
- 카운트·주자 상황마다 다른 배합
- 의도적 볼(waste)로 read 정보换 vs fishing으로 헛스윙

### 4.2 패턴 노출도 (Pattern Exposure)

별도 게이지가 아니라 **`exposed` 불리언 + signals[]`** 로 표현:

- `exposed = suspicion ≥ 58 && signals.length > 0`
- UI: 「타자가 ○○을 의식합니다」(signals[0])
- 로그: 「타자가 결정구 패턴을 의식하기 시작합니다」
- 3구 이상 간격으로 경고 재표시 (`lastWarningAt`)

**기획 의도**: 58%는 “읽히기 시작” 체감점. 72%는 “역으로 속임” 전환점.

### 4.3 타자 적응도 (Adaptation)

명시적 “적응도” 스탯은 없고, 아래가 **적응력**을 대표:

| 요소 | 역할 |
|------|------|
| `mind.patternLearn` | 의심 증가 속도 (적응형 1.26) |
| `memory.span` | 기억하는 투구 수 |
| `reactive` tendency | suspicion 50+ 시 contact +0.08 |
| `repeatedPenalty` | 같은 category 연속 투구 시 타자 유리 (+0.07 per repeat) |
| 스테이지 타자 스탯 boost | S2 +14~22, S3 +28~36 |

### 4.4 멘탈 변화

#### 투수 멘탈

```
주자 있을 때:
  pressurePenalty = max(0, 72 − 멘탈) × 주자수 × 0.18  (제구)
  pitcherEffectiveStat = stat − max(0, 68−멘탈)×0.08×(주자있음)

약점 태그:
  pressure_wobble → pressurePenalty +4
  walk_risk (볼넷 2+) → command −4
  full_count_wobble (3-2 × 3회) → command −5
```

#### 타자 “멘탈” (마음·성향)

- **정직형**: 반응 reliability 86% — read UI 신뢰
- **교활형**: fakeRate 20%, reliability 42%
- **거짓 단서** 확률:

```
falseClueChance = clamp(
  mind.fakeRate + suspicion/520 + (reverseRead ? 0.08 : 0),
  0.02, 0.42
)
```

→ 발생 시 **잘못된 계열** readScores +0.55, 실제 계열 −0.12

#### 노림수 추정 바 (`readScores`)

매 투구 후 갱신 (플레이어용 정보):

| 결과 | 점수 변화 |
|------|-----------|
| 스윙 + timing > 0.68 | 일치 +1.35 / 불일치 +0.75 |
| foul | selected +0.42 |
| ball (존 밖) | selected −0.16 |
| calledStrike | selected −0.22, 타 계열 +0.13 |
| timing 「너무 빠름」 | fast +0.8 |
| timing 「늦음」 | breaking/offspeed +0.46 |
| 「완전히 속음」 | selected −0.3 |
| falseClue | fake 계열 +0.55 |
| waste 볼 | selected +0.22 |

각 계열 clamp(0.2, 8).

#### 노림수 추천 (`buildRecommendation`)

```
accuracy = clamp(0.28 + 예측/125 − (suspicion≥70 && tricky ? 0.12 : 0), 0.35, 0.88)
guessedTarget = chance(accuracy) ? realTarget : weightedOther
recommendedCategory = counterCategory(real or guessed)
```

counter 테이블:

| 노림 | 우선 counter |
|------|--------------|
| fast | breaking → offspeed → fast |
| breaking | fast → offspeed → breaking |
| offspeed | fast → breaking → offspeed |

## 5. 결과 판정 공식

### 5.1 판정 트리

```
                    ┌─ NO ─→ inZone? ─ YES → calledStrike (루킹)
                    │              └ NO  → ball
              swung?
                    │
                    └─ YES ─→ contactProbability?
                              ├─ NO  → swingingStrike (헛스윙)
                              └─ YES ─→ foulProbability && timing<0.78?
                                        ├─ YES → foul (2스트 전만 S+1)
                                        └─ NO  → contactQuality → inPlay 분기
```

### 5.2 구위 (quality)

```
quality = clamp(
  pitchQuality(pitch)           // 구속/변화/제구 + 구종 base
  + (commandScore − 60) × 0.08
  + special + mind + profile + tag,
  20, 99
)

pitchQuality:
  fast:     구속×0.52 + 제구×0.34 + pitch.speed×0.18 + pitch.control×0.16
  non-fast: 변화×0.55 + 제구×0.34 + movement×0.18 ...
```

### 5.3 스윙 확률

```
swingProbability =
  (inZone ? 0.46 : 0.16)
  + approach 보정
  + (targetMatch ? 0.20 : −0.09)
  + countSwingAdjustment
  + 구종특수 + mind + tag
  + (centerMistake ? 0.11~0.18 : 0)
  + (존 밖 ? chase + tag.chase − 선구/230 : 0)

swung = random() < swingProbability
```

### 5.4 타이밍 (스윙 시)

```
deception = targetMatch ? −0.12 : +0.16
repeatedPenalty = max(0, 동일category연속−1) × 0.07

timingBase =
  0.34 + 타자.예측/180
  + (targetMatch ? 0.20 : −0.11)
  − quality/360
  − repeatedPenalty
  + deception

timingValue = clamp(timingBase + random(−0.17~+0.17), 0.05, 0.98)
```

**타이밍 라벨** (`timingText`):

| value | 일반 | 노림 일치 시 |
|-------|------|--------------|
| > 0.82 | 완벽 | 완벽 |
| > 0.66 | 좋음 | 좋음 |
| > 0.48 | 불안정 | 불안정 |
| ≤ 0.48 | 늦음/빠름/속음 | 밀림 / **완전히 속음** |

### 5.5 컨택 확률

```
contactProbability =
  0.14
  + 타자.컨택/170
  + timingValue × 0.34
  − quality/260
  + (targetMatch ? 0.09 : −0.08)
  + repeatedPenalty
  + special/mind/tag/contactBonus
  + (완전히 속음 ? −0.22 : 0)
```

### 5.6 파울 확률

```
foulProbability =
  0.34
  + |0.58 − timingValue| × 0.28
  + (2스트 ? 0.12 : 0)
  + mind.foul

→ chance(foulProbability) && timingValue < 0.78 → foul
→ 2스트 이후 foul은 S 증가 없음
```

### 5.7 contactQuality (인플레이 품질)

```
contactQuality =
  타자.컨택 × 0.28
  + 타자.파워 × 0.48
  + timingValue × 42
  − quality × 0.33
  + (targetMatch ? 14 : −10)
  + repeatedPenalty × 70
  + special/mind/tag/profile/centerBonus
  + (완전히 속음 ? −30 : 0)
  + random(−12, +16)
```

### 5.8 인플레이 결과 분기

| contactQuality | 결과 | 추가 조건 |
|----------------|------|-----------|
| 「완전히 속음」 | GROUND OUT | CQ>48 & 10% → **TEXAS HIT** (1루타) |
| > 82 | **HOME RUN** | |
| > 67 | **2루타** | |
| > 52 | **1루타** | |
| ≤ 50 & 1루 주자 & out<2 | **병살** chance | 0.14 + breaking +0.12 + low +0.10 + tag |
| 약타구 | **실책** | 1.5~5.5% (땅볼·주자·멘탈) |
| CQ < 38 or 투심 or low row | GROUND OUT | |
| 그 외 | FLY OUT | |

### 5.9 B/S/O·볼넷·삼진 (야구 규칙)

| 결과 | B/S/O |
|------|-------|
| ball | B+1, 4볼 → **WALK** |
| calledStrike / swingingStrike | S+1, 3S → **삼kin** (루킹/헛스윙 구분 → 보상 차등) |
| foul | S+1 (**2스트 미만일 때만**) |
| inPlay out / DP | out+1 or +2 |

## 6. 성장·보상 구조

### 6.1 성장 축 3종

```
① 투수 스탯 (5종)     — 매 타석 보상
② 구종 control        — 해당 구종 숙련 보상
③ 태그 (base/bonus/weakness) — 스테이지 보상 + 조건 공개
```

### 6.2 타석 보상 트리거

| 트리거 | rewardReason | 풀 크기 |
|--------|--------------|---------|
| 스윙 삼kin | 「스윙삼kin 보상」 | 기본 |
| 루킹 삼kin | 「루킹삼kin 보상」 | 기본 |
| 병살 | 「병살 보너스」 / 「변화구 병살 보너스」 | **확장** (+변화/멘탈) |
| 보스 아웃 | 「보스 타자 제압」 등 | **확장** (+제구/예측) |
| 땅볼/플라이 아웃 | — | **보상 없음** |
| 안타·홈런·볼넷 | — | **보상 없음** (실점 페널티) |

→ **잘 던져서 아웃 잡을수록** 성장. 실점은 보상 없음.

### 6.3 일반 보상 3택1 (`generateRewardChoices`)

**기본 풀** (그 중 3개 랜덤):

| type | 내용 | amount |
|------|------|--------|
| stat | 제구/구속/변화/멘탈/예측 | +1~2 |
| pitch | 방금 구종 control | +3~5 |
| newPitch | 미보유 구종 1개 | repertoire < 5 |

병살/보스 시 추가 stat 항목이 풀에 합류 → **같은 3택1이지만 고급 선택지 비율↑**.

### 6.4 스테이지 보상

스테이지 클리어 (3/5/7이닝 + 실점 한도):

1. 실점·주자·이닝·타순 **리셋**
2. 새 9명 라인업 (스탯 boost)
3. **보조 태그 3택1** (`generateStageTagChoices`)
   - `bonus` 타입만, 중복·보유 제외
   - 프로필 일치 태그 가중 ×3

| 스테이지 | 이닝 | 실점 한도 | 타자 boost |
|----------|------|-----------|------------|
| 1 | 3 | 5 | — |
| 2 | 5 | 4 | +14~22 |
| 3 | 7 | 3 | +28~36 |

### 6.5 한 경기 성장 곡선

```
[선발] 프로필 + baseTag + hiddenWeakness×2 + 3~5구종
   ↓ 타석 보상 (삼kin/병살/보스)
   stat↑, pitch control↑, 구종 추가
   ↓ 스테이지1 클리어
   bonusTag #1
   ↓ 스테이지2 (강한 타자)
   stat↑, bonusTag #2
   ↓ 스테이지3 (7이닝 마라톤)
   bonusTag 없음(최종) — 빌드 완성도로 승부
```

**투수·구종·태그는 경기 내내 유지**. 스테이지만 리셋.

### 6.6 태그가 판정에 미치는 방식

| 태그 | effects → 적용 시점 |
|------|---------------------|
| 2스트 특화 | twoStrikeQuality +4, contact −0.05 |
| 헛스윙 유도 | whiff +0.06, chase +0.04 |
| 땅볼 유도 | groundball +0.12, low CQ −5 |
| 낮은 코스 장인 | lowZoneControl +6 |
| 장타 위험 (약점) | centerMistake homerun +8 |
| 멘탈 회복 | pressureReduce +4 |

## 7. 빌드 시너지 예시

내부 재미 검증용 **의도된 빌드 archetype**:

### 7.1 「헛스윙 장인」

- 선발: 변화구 장인 + 슬라이더/커브
- 보상: 헛스윙 유도, 2스트 특화, 변화↑
- 플레이: 바깥 fishing 볼 → 2스트 slider/curve
- 리스크: pattern 노출 시 exposed → contact↑

### 7.2 「제구 뱀」

- 선발: 제구형 + command 태그
- 보상: 제구↑, 낮은 코스/바깥 제구, 멘탈 회복
- 플레이: 코너 strike, walkHunter 상대 볼넷 유도
- 리스크: 볼넷 2 → walk_risk 약점 공개

### 7.3 「속도 + 완급」

- 선발: 파워 피처 + 체인지/스플리터
- 보상: 구속↑, fast control, 체인지 숙련
- 플레이: fast → offspeed 교차, suspicion 관리
- 리스크: 같은 family 반복 시 suspicion +7

### 7.4 「예측 심리전」

- 선발: 균형형, 예측 높은 롤
- 보상: 예측↑, 멘탈, 다양 구종
- 플레이: 추천 UI 88%까지, counter 구종
- 리스크: tricky + suspicion 70+ → 추천 −12%p
